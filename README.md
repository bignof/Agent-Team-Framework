# Agent Team 项目模板

基于 Claude Code 的多角色协作开发框架，支持从市场调研到部署上线的完整开发流程。

## 快速启动

通过 `@角色名` 召唤对应 Agent：

### 产品阶段
| 角色 | 说明 |
|------|------|
| `@市场调研` | 行业趋势分析、竞品扫描、用户画像 |
| `@产品经理` | PRD 编写、需求建模、验收标准 |
| `@UI/UX 设计师` | 交互流程图、全逻辑 HTML 原型 |

### 技术阶段
| 角色 | 说明 |
|------|------|
| `@架构师` | 技术选型、系统架构设计、API 规范 |
| `@AI 工程师` | RAG、Fine-tuning、Prompt 工程 |
| `@后端工程师` | API 实现、数据库设计 |
| `@前端工程师` | React/Vue 组件、响应式页面 |
| `@测试工程师` | 测试策略、E2E 测试、测试报告 |
| `@DevOps` | CI/CD、容器化、监控告警 |

## 开发流程

```
1. @市场调研 → 输出市场调研报告
2. @产品经理 → 输出 PRD 文档
3. @UI/UX 设计师 → 输出交互原型
4. @架构师 → 输出技术架构设计
5. @AI 工程师 → 输出 AI 技术方案 (如需要)
6. @后端工程师 → 实现 API 接口
7. @前端工程师 → 实现前端页面
8. @测试工程师 → 输出测试报告
9. @DevOps → 部署上线
```

## 文档规范

所有文档存储在 `.claude/doc/` 目录下：

```
.claude/doc/
├── PROJECT_INDEX.md          # 项目文档索引
├── 01_Product_Design/        # 产品定义
│   └── prototypes/           # 交互原型
├── 02_Architecture/          # 技术架构
├── 03_API_Contract/          # 接口规范
├── 04_Test_Reports/          # 测试报告
└── 05_DevOps/                # 运维部署
```

## 项目结构

```
.
├── .claude/
│   ├── agents/               # Agent 角色定义
│   ├── doc/                  # 项目文档
│   └── settings.local.json   # 本地设置
├── CLAUDE.md                 # Agent 编排配置
├── README.md                 # 项目说明
└── .gitignore
```

## 使用说明

1. **召唤 Agent**: 在对话中输入 `@角色名` 即可召唤对应 Agent
2. **查看文档**: 所有生成的文档会自动存放在 `.claude/doc/` 对应目录
3. **更新索引**: 任务完成后，Agent 会自动更新 `PROJECT_INDEX.md`

## 自定义 Agent

在 `.claude/agents/` 目录下创建新的 `.md` 文件定义新角色：

```markdown
### 1. 角色定义

你是一名...

### 2. 核心任务

1. 任务 1
2. 任务 2

### 3. 交付规范

- 文件路径：...
- 命名规范：...

### 4. 协作协议

- 前置输入：...
- 输出反馈：...
```
