## 2. 功能清单

> 💡 **填写说明**：
> - **P0**：核心功能，缺此产品无法上线（通常不超过 5 个）
> - **P1**：重要功能，影响用户体验但可延后 1-2 个版本
> - **P2**：锦上添花功能，有时间再做
> - **复杂度**：开发难度评估（高/中/低）
> - **依赖**：前置功能或外部依赖

| 优先级 | 功能模块 | 功能描述 | 复杂度 | 依赖 |
|--------|---------|---------|--------|------|
| P0 | | | 高/中/低 | |
| P1 | | | 高/中/低 | |
| P2 | | | 高/中/低 | |

**示例行**：
| P0 | 用户注册登录 | 支持邮箱注册、密码登录、忘记密码找回 | 中 | 无 |
| P1 | 第三方登录 | 支持 Google、GitHub OAuth 登录 | 低 | P0 完成 |

## 5. 用户故事 (User Stories)

> 💡 **填写说明**：
> - 格式：**作为** [角色]，**我想要** [功能]，**以便** [价值]
> - 每个用户故事必须有验收标准（AC），AC 应该是可测试的是/否问题
> - 优先级 P0/P1 的功能至少需要 2-3 条 AC

### 4.1 [功能模块 A]

> 💡 **示例**：
> ### 4.1 用户登录
> - **作为** 注册用户，**我想要** 使用邮箱和密码登录，**以便** 访问我的个人账户
>   - 验收标准：
>     - [ ] AC1: 输入正确的邮箱和密码后，成功跳转到仪表盘
>     - [ ] AC2: 密码错误时，显示明确的错误提示"邮箱或密码错误"
>     - [ ] AC3: 连续 5 次失败后，账号锁定 15 分钟

- **作为** [用户角色]，**我想要** [功能]，**以便** [价值]
  - 验收标准：
    - [ ] AC1
    - [ ] AC2

### 4.2 [功能模块 B]
- **作为** [用户角色]，**我想要** [功能]，**以便** [价值]
  - 验收标准：
    - [ ] AC1

## 6. 非功能需求

> 💡 **填写说明**：非功能需求应该是可量化的指标，避免"快速"、"安全"等模糊表述。

### 5.1 性能指标
> 💡 **示例**：
> - 首页首屏加载时间 < 2s（4G 网络，中端设备）
> - API P95 响应时间 < 500ms
> - 支持并发用户数 1000+

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| 页面加载时间 | < 2s | 首屏渲染时间 |
| API 响应时间 | < 500ms | P95 延迟 |

### 5.2 安全性
> 💡 **示例**：
> - 所有 API 必须使用 HTTPS
> - 密码使用 bcrypt 加密存储
> - JWT token 有效期不超过 7 天
