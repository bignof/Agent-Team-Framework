- 前端完成核心页面开发（80% 以上）

- QG-02（架构设计）已通过

**联调规则**：

- 前后端代码实现完成后，自动进入联调阶段
- 架构师负责创建联调报告 `integration_report_[项目].md`

## 2. 联调执行清单

> 联调由架构师角色主导，按以下清单逐项完成。

**架构师角色执行项**：

- 创建联调报告文档 `integration_report_[项目].md`
- 切换前端代码到真实 API 环境
- 逐一验证接口连通性（参考 Section 4 清单）
- 修复发现的接口问题（数据格式、CORS、认证等）
- 确保 API 文档与实际实现一致
- 记录联调问题和进度
- 联调完成后建议用户召唤 @测试工程师

## 3. 联调环境配置

### 3.1 环境信息

| 环境          | URL                     | 用途     |
| ------------- | ----------------------- | -------- |
| 前端开发环境  | `http://localhost:3000` | 前端调试 |
| 后端 API 环境 | `http://localhost:8000` | 后端 API |

### 3.2 CORS 配置

**后端需要配置的 CORS**：

```javascript
// Express 示例
app.use(
  cors({
    origin: 'http://localhost:3000', // 前端地址
    credentials: true,
    allowedHeaders: ['Content-Type', 'Authorization'],
  }),
)
```

## 4. 联调步骤

### 4.1 接口连通性验证

**检查清单**：

- [ ] GET 请求可以正常获取数据
- [ ] POST 请求可以正常提交数据
- [ ] 请求头正确（Content-Type、Authorization）
- [ ] CORS 配置正确（无跨域错误）

### 4.2 数据格式验证

**检查清单**：

- [ ] 响应数据格式与 API 合同一致
- [ ] 字段类型正确（string、number、boolean）
- [ ] 分页数据格式正确
- [ ] 日期格式正确（ISO 8601）

### 4.3 错误处理验证

**检查清单**：

- [ ] 400/401/403/404/500 错误正确处理
- [ ] 错误响应格式统一

### 4.4 认证鉴权验证

**检查清单**：

- [ ] 登录接口正常（获取 Token）
- [ ] Token 正确传递（Authorization header）
- [ ] 受保护接口需要认证

### 4.5 核心业务流程验证

**检查清单**：

- [ ] 核心业务流程可以完整跑通
- [ ] 数据状态变更正确

## 5. 联调问题跟踪

### 5.1 问题分级

| 级别   | 定义                     | 处理时效                   |
| ------ | ------------------------ | -------------------------- |
| **P0** | 阻塞性问题，无法继续联调 | 立即修复，最高优先级       |
| **P1** | 严重问题，影响核心功能   | 优先修复，在联调完成前解决 |
| **P2** | 一般问题，不影响主要流程 | 可记录后修复               |

### 5.2 问题跟踪表

| 编号 | 问题描述 | 接口/模块 | 级别     | 状态                       |
| ---- | -------- | --------- | -------- | -------------------------- |
| 001  |          |           | P0/P1/P2 | ☐ 待修复/☐ 修复中/☐ 已修复 |
