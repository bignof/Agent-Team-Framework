# [阶段名称] 问题跟踪

> 本文件用于记录其他角色对本阶段文档的疑问、待澄清事项和修改请求。
> 由提问方创建/追加问题，由本阶段负责角色解答。

---

## 使用规则

1. **问题归属原则**：问题记录在**被质疑文档所在的目录**，而非提问者所在目录
   - 例：架构师对 PRD 有疑问 → 记录在 `01_Product_Design/ISSUES.md`
   - 例：前端对 API 有疑问 → 记录在 `03_API_Contract/ISSUES.md`
2. **提问方职责**：创建或追加问题条目，标注优先级和阻塞状态
3. **解答方职责**：用户切换到对应角色后，读取 ISSUES.md 并逐条解答
4. **闭环管理**：问题解答后必须更新状态为 `✅ 已解决`，并注明对文档的影响

---

## 问题状态说明

| 状态      | 含义                                 |
| --------- | ------------------------------------ |
| 🔴 待解答 | 新提出的问题，等待负责角色解答       |
| 🟡 讨论中 | 已有初步回复，但需要进一步确认       |
| ✅ 已解决 | 问题已解答，相关文档已更新（如需要） |
| ⏸️ 暂缓   | 用户决定延后处理                     |

## 优先级说明

| 优先级      | 含义                                    |
| ----------- | --------------------------------------- |
| **P0-阻塞** | 阻塞当前角色工作，必须先解决才能继续    |
| **P1-重要** | 不阻塞当前工作，但影响后续设计/实现质量 |
| **P2-建议** | 优化建议或边界情况确认，可稍后处理      |
