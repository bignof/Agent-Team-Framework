## 2. 迁移脚本规范

### 2.1 命名规范

```
migration_[描述]_[日期].sql
migration_add_user_phone_20260301.sql
```

### 2.2 脚本内容要求

**必须包含**：
- 迁移说明（注释）
- 迁移操作（UP）
- 回滚操作（DOWN）
- 数据备份（如涉及数据修改）

**示例**：
```sql
-- Migration: 添加用户手机号字段
-- Date: 2026-03-01
-- Author: @后端工程师
-- Description: 在 users 表添加 phone 字段，用于短信验证码

-- UP (迁移操作)
ALTER TABLE users ADD COLUMN phone VARCHAR(20);
ALTER TABLE users ADD INDEX idx_phone (phone);

-- DOWN (回滚操作)
DROP INDEX idx_phone ON users;
ALTER TABLE users DROP COLUMN phone;
```

### 2.3 数据备份要求

**涉及数据修改时必须备份**：
```sql
-- 备份原数据
CREATE TABLE users_backup_20260301 AS SELECT * FROM users;

-- 执行修改
ALTER TABLE users MODIFY COLUMN email VARCHAR(255) NOT NULL;

-- 验证后可删除备份（一周后）
-- DROP TABLE users_backup_20260301;
```

## 4. 迁移风险控制

### 4.1 风险等级

| 等级 | 定义 | 处理方式 |
|------|------|---------|
| **低风险** | 新增表、新增字段（不影响现有数据） | 直接执行 |
| **中风险** | 修改字段类型、删除字段 | 备份后执行 |
| **高风险** | 核心数据修改、大批量数据操作 | 用户确认后执行，准备回滚方案 |

### 4.2 回滚方案

**必须准备回滚方案的情况**：
- 中风险迁移
- 高风险迁移

**回滚方案内容**：
1. 回滚步骤
2. 回滚时间预估
3. 回滚后的数据一致性保证
