# 测试报告目录

本目录存放测试相关报告，由 @测试工程师 负责维护和更新。

## 文档类型

| 文档类型     | 命名规范                            | 说明                             |
| ------------ | ----------------------------------- | -------------------------------- |
| 功能测试报告 | `test_report_[功能名称]_[日期].md`  | 功能测试结果、覆盖率、缺陷清单   |
| 集成测试报告 | `test_report_integration_[日期].md` | API 集成、数据库、第三方服务测试 |
| E2E 测试报告 | `test_report_e2e_[日期].md`         | 用户旅程、跨浏览器兼容性测试     |
| 性能测试报告 | `test_report_performance_[日期].md` | 负载测试、压力测试、性能基准     |

## 测试策略

测试金字塔：

- **单元测试**: 核心业务逻辑覆盖率 > 80%
- **集成测试**: API 接口、数据库事务、第三方服务
- **E2E 测试**: 用户旅程、兼容性、可访问性

## 质量卡点

- **QG-03**: 代码实现 → 测试（联调完成确认）
- **QG-04**: 测试 → 部署（5 项检查，含“严重 Bug 清零”）

## 使用说明

1. @测试工程师 完成测试后必须生成测试报告
2. 测试报告必须包含通过率、缺陷清单、修复建议
3. P0/P1 级别 Bug 清零后才能通过 QG-04
4. 所有文档创建后必须更新 `../PROJECT_INDEX.md`
