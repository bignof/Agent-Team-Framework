# 技术架构文档目录

本目录存放技术架构相关文档，由 @架构师 和 @AI 工程师 共同维护。

## 文档类型

| 文档类型 | 命名规范 | 负责角色 | 说明 |
|---------|---------|---------|------|
| 架构设计 | `arch_[项目简称]_[文档类型]_v[版本号].md` | @架构师 | 技术选型、系统架构图、模块划分 |
| 配置规范 | `config_[项目简称]_v[版本号].md` | @架构师 | 中间件清单、Docker Compose、环境变量模板 |
| AI 技术方案 | `ai_design_[功能名称]_[日期].md` | @AI 工程师 | RAG 架构、Prompt 设计、模型选型 |

## 配置传递（重要）

@架构师 完成架构设计后必须创建 `config_[项目简称]_v1.0.md`，包含：

1. **中间件清单** - 开发环境配置（给 @后端工程师）
2. **Docker Compose** - 一键启动开发环境
3. **环境变量模板** - .env.development / .env.staging / .env.production

@后端工程师 和 @DevOps 必须读取配置文档获取配置信息。

## 质量卡点

- **QG-03**: 架构设计 → 代码实现（6 项检查，含"配置文档已创建"）

## 使用说明

1. @架构师 完成架构设计后必须创建配置文档
2. 配置文档必须通过 QG-03 检查
3. 所有文档创建后必须更新 `../PROJECT_INDEX.md`
