# API 接口规范目录

本目录存放 API 接口规范文档，由 @架构师 和 @后端工程师 共同维护。

## 文档类型

| 文档类型 | 命名规范 | 负责角色 | 说明 |
|---------|---------|---------|------|
| API 规范 | `api_[模块名]_v[版本号].md` | @架构师/@后端 | OpenAPI/Swagger 格式的接口规范 |
| 错误码规范 | `error_codes_v[版本号].md` | @架构师 | 统一错误码定义和说明 |

## 维护规范

1. **架构师负责**: 创建初始 API 规范，定义接口格式、错误码、版本控制策略
2. **后端工程师负责**: 实现 API 后更新文档，确保文档与代码一致
3. **前后端协作**: API 规范作为前后端开发的契约依据

## 质量卡点

- **QG-03**: 架构设计 → 代码实现（包含"API 规范已定义"检查）
- **QG-04**: 代码实现 → 测试（包含"API 文档与实现一致"检查）

## 使用说明

1. @架构师 在架构设计阶段创建 API 规范
2. @后端工程师 实现 API 后必须更新文档
3. 所有文档创建后必须更新 `../PROJECT_INDEX.md`
