# 交互原型目录

本目录存放 UI/UX 设计师创建的可交互 HTML 原型。

## 设计规范

### 核心原则

- ❌ **禁止假按钮**: 所有可见按钮必须有点击响应和实际功能
- ❌ **禁止假页面**: 所有页面链接必须能跳转到真实可操作的页面
- ✅ **完整交互**: 用户点击任何元素都必须有明确的反馈
- ✅ **模拟数据闭环**: 使用 localStorage 实现完整的 CRUD 操作
- ✅ **基于线框图**: 高保真原型应遵循产品经理定义的页面布局

### 技术要求

- **技术栈**: HTML5 + Tailwind CSS (CDN) + Lucide Icons (CDN) + Alpine.js / Vanilla JS
- **交付形态**: 交互原型允许单文件或多文件形式
  - 推荐采用“每个版本一个文件夹”的方式管理（例如：`proto_[项目简称]_v1.0/`）
  - 每个版本文件夹必须提供统一入口 `index.html`，用于预览与联调对齐
  - 如需拆分资源（CSS/JS/图片等）应保持体量克制，避免引入不必要的构建工具
- **多页面**: 可使用 JS 路由或多 HTML 页面实现导航

### 交互逻辑要求

1. **状态管理**: 登录状态、加载状态、空状态、错误状态
2. **动态交互**: 表单验证、提交反馈、列表操作、弹窗抽屉
3. **数据模拟**: localStorage 存储、CRUD 操作、实时更新
4. **响应式**: 桌面端和移动端适配

## 文件命名规范

| 文件类型         | 命名格式                    | 示例                |
| ---------------- | --------------------------- | ------------------- |
| 版本目录         | `proto_[项目简称]_v[版本]/` | `proto_myapp_v1.0/` |
| 主入口           | `index.html`                | `index.html`        |
| 资源目录（可选） | `assets/`                   | `assets/app.js`     |

## 使用说明

1. 打开对应版本目录下的 `index.html` 访问主入口
2. 所有按钮和链接都必须有实际功能
3. 原型完成后必须经过 @产品经理评审确认
4. 评审通过后才能在父目录更新设计说明文档
