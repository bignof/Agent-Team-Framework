# 交互原型目录

本目录存放 UI/UX 设计师创建的可交互 HTML 原型。

## 设计规范

### 核心原则
- ❌ **禁止假按钮**: 所有可见按钮必须有点击响应和实际功能
- ❌ **禁止假页面**: 所有页面链接必须能跳转到真实可操作的页面
- ✅ **完整交互**: 用户点击任何元素都必须有明确的反馈
- ✅ **模拟数据闭环**: 使用 localStorage 实现完整的 CRUD 操作

### 技术要求
- **技术栈**: HTML5 + Tailwind CSS (CDN) + Lucide Icons (CDN) + Alpine.js / Vanilla JS
- **单文件**: 每个页面是一个独立的 HTML 文件，所有 CSS/JS 内联
- **多页面**: 使用 JS 路由或锚点实现多页面导航

### 交互逻辑要求
1. **状态管理**: 登录状态、加载状态、空状态、错误状态
2. **动态交互**: 表单验证、提交反馈、列表操作、弹窗抽屉
3. **数据模拟**: localStorage 存储、CRUD 操作、实时更新
4. **响应式**: 桌面端和移动端适配

## 文件命名规范

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 主入口 | `proto_[项目简称]_index.html` | `proto_myapp_index.html` |
| 分页面 | `proto_[项目简称]_[页面名].html` | `proto_myapp_login.html` |

## 使用说明

1. 打开 `proto_[项目简称]_index.html` 访问主入口
2. 所有按钮和链接都必须有实际功能
3. 原型完成后必须在父目录更新设计说明文档
