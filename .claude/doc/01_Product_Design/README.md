# 产品定义和设计文档目录

本目录存放产品设计相关文档，由 @市场调研、@产品经理、@UI/UX 设计师 共同维护。

## 文档类型

| 文档类型 | 命名规范 | 负责角色 | 说明 |
|---------|---------|---------|------|
| 市场调研 | `market_research_[项目简称]_[日期].md` | @市场调研 | 竞品分析、用户画像、市场趋势 |
| PRD | `prd_[项目简称]_v[版本号].md` | @产品经理 | 产品需求文档（含页面流程和数据结构） |
| 设计说明 | `ux_design_[项目简称]_[版本].md` | @UI/UX 设计师 | 用户流程图、设计系统规范 |

## 子目录

| 子目录 | 说明 |
|-------|------|
| `prototypes/` | 交互原型 HTML 文件（完整可交互，无假按钮） |

## 质量卡点

- **QG-02**: 产品需求 → 架构设计（7 项检查）
- **QG-02.5**: 产品需求 → 交互原型（6 项检查）

## 使用说明

1. @产品经理 完成 PRD 后必须通过 QG-02 和 QG-02.5 检查
2. @UI/UX 设计师必须基于 PRD 创建完整可交互的原型
3. 所有原型必须支持完整的 CRUD 操作（使用 localStorage）
4. 所有文档创建后必须更新 `../PROJECT_INDEX.md`
