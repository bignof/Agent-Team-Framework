# 产品定义和设计文档目录

本目录存放产品设计相关文档，由 @市场调研、@产品经理、@UI/UX 设计师 共同维护。

## 文档类型

| 文档类型 | 命名规范 | 负责角色 | 说明 |
|---------|---------|---------|------|
| 市场调研 | `market_research_[项目简称]_[日期].md` | @市场调研 | 竞品分析、用户画像、市场趋势 |
| PRD（含线框图） | `prd_[项目简称]_v[版本号].md` | @产品经理 | 产品需求文档（含页面流程和**页面线框图**） |
| 设计说明 | `ux_design_[项目简称]_[版本].md` | @UI/UX 设计师 | 用户流程图、设计系统规范 |

## 子目录

| 子目录 | 说明 |
|-------|------|
| `prototypes/` | 交互原型 HTML 文件（完整可交互，无假按钮） |

## 质量卡点

- **QG-01**: 项目规划 → 产品需求（**用户评审**）
- **QG-02**: 产品需求 → 架构设计/原型（**用户评审 PRD+ 线框图**，10 项检查）
- **QG-02.5**: 交互原型 → 技术确认（**用户评审原型**，10 项检查）
- **QG-03**: 架构设计 → 代码实现（**用户评审**，6 项检查）
- **QG-04**: 代码实现 → 测试
- **QG-05**: 测试 → 部署（**用户评审**，5 项检查）

## 使用说明

1. @产品经理 完成 PRD 后必须绘制**页面线框图**，并通过 QG-02 检查（**用户评审确认**）
2. @架构师在 QG-02 通过后即可开始架构设计，**无需等待 QG-02.5**
3. @UI/UX 设计师在 QG-02 通过后开始原型设计，完成后通过 QG-02.5 检查（**用户评审确认**）
4. @前端工程师在 QG-02.5 通过后才能开始实现
5. 所有文档创建后必须更新 `../PROJECT_INDEX.md`
